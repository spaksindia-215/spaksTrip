Insurance GetBooking Details

This method used to get the booking information based on Booking ID or Policy No or Trace ID It is also used to retrieve the booking status which might be Hold, Booked, failed, cancelled, etc.

Service URL (REST) to be used:

https://InsuranceBE.tektravels.com/InsuranceService.svc/rest/GetBookingDetail

Insurance GetBooking Request
Level Element Format Comments Mandatory/ Optional
1 EndUserIp String IP Address of the end user Mandatory
2 TokenId String Token ID Mandatory
(Token Id to be echoed back from Authenticate response)
3 BookingId Integer Unique booking id of the successful booking Mandatory
(will always be greater than 0)
Insuance Getbookingdetail Response
Level Element Format Comments Mandatory/ Optional
1 Itinerary Object Contain the insurance policy details Mandatory
2 BookingId Integer Unique booking id of the successful policy Mandatory
3 InsuranceId Integer Unique insurance id of the successful policy Mandatory
4 PlanType Define insurance is single trip or AnnualMultiTrip (Ex: SingleTrip = 1, AMT = 2) Mandatory
5 PlanName Name of the plan Mandatory
6 PlanDescription String Description of the plan Mandatory
7 PlanCoverage Enumeration Define whether the plan coverage area is US, NON us and India (Ex: US = 1 Non_US = 2, WorldWide = 3 India = 4 Asia = 5 Canada = 6 Australia = 7 SchenegenCountries = 8) Mandatory
8 CoverageDetails Array Mandatory
8.1 Coverage Define the Insurance cover area ( Ex: 24/7 Worldwide Medical Assistance, Travel Assistance, ravel Insurance, Dental Cover Mandatory
8.2 SumInsured Define the service wise cover amount (Ex Dental Cover -$250, Personal Accident-$10000) Mandatory
9 PlanCategory Define the category of plan whether its Domestic, Overseas and studentoversea plan DomesticTravelPolicy = 1, OverseasTravelInsurance = 2, StudentOverseasInsurance = 3, SchengenOverseasInsurance = 4, InboundTravelPolicy = 5, CancellationInsurance = 6 Mandatory
10 Passenger Info Array Mandatory
10.1 PolicyNo Integer Unique policy number of the successful booking Mandatory
10.2 MaxAge Define max age of passenger for which policy is valid Mandatory
10.3 MinAge Define min age of passenger for which policy is valid Mandatory
10.4 Title Enumeration Title of the passenger (Possible values can be Mr, Mrs, Miss, Ms,SHRI,SMT) Mandatory
10.5 FirstName String First name of the insured passenger Mandatory
10.6 LastName String Last name of the insured passenger Mandatory
10.7 Gender String Gender Mandatory
10.8 DOB Date Time DOB of the insured passenger Format: (dd/mm/yyyy) Mandatory

10.10 RelationShipToInsured String RelationShipToInsured(Ex:Self) Mandatory
